⚠ Ranking methodology — what is exact vs. approximated
  • Exact: Rank-sum over the 12 published metrics (8 lesionwise DSC/NSD for ET/TC/WT/RC + 4 small-instance F1). Higher score → better rank on each metric. Missing metrics are not penalised (not set to worst value), per the challenge rules. The best submission per team is selected by highest mean across the same 12 published ranking metrics used for the final leaderboard.
  • Approximated / missing: The full DELPHI protocol requires per-case metric arrays to run 500,000 pairwise permutation tests and produce a statistical tiering (p-value upper-triangular matrix). The Synapse public API only exposes per-submission aggregate means. Statistical tiers and p-values therefore cannot be reproduced and are not shown. What you see is the rank ordering, which is the first step of DELPHI and is exact given the public data.
  • To reproduce exact tiers: obtain per-case CSV from the challenge organizers.
